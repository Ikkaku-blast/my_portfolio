About Me

写真

    黒沼慎太郎

    大阪大学大学院基礎工学研究科システム創成専攻に所属しています。大学でゲーム制作を始め、ルール設計によってプレイヤーの体験や感情が変化する点に魅力を感じ、ゲームプランナーを志望するようになりました。企画・UI制作・実装に取り組み、考えた体験を自分の手で形にする力を磨いています。チーム制作では役割分担や情報共有の仕組みづくりも行い、新聞や教育・お笑いなど幅広い関心領域から企画の引き出しを増やすことを意識しています。

    HTML/CSS JavaScript Python Unity

    Strengths

    体験を設計する

    作品ごとに、まずプレイヤーへ届けたい体験と面白さの条件を言語化します。そのうえで、ルール・UI・演出が同じ方向を向くように設計し、制作全体の判断基準を明確にしています。

    「ミスった、ドーナツ」では、「直感的に理解できること」と「短時間で判断する緊張感」を方針に設定しました。注文UI、制限時間の色変化、商品選択ルールをその方針に合わせ、プレイヤーが状況を瞬時に読み取り判断できる体験を狙いました。

    チーム制作を進める仕組みづくり

    チーム制作では、役割や進行状況が曖昧なままだと判断が遅れるため、Notionで担当・優先度・進捗を共有しました。各メンバーが次に取り組む作業を判断しやすくし、遅れている作業にも早めに支援を回せる状態を作りました。

    また、仕様書で世界観・ルール・面白さの方向性を共有したうえで、匿名のアイデア掲示板を用意しました。会議中に出しにくい案や後から思いついたネタも集められるようにし、方向性を保ちながら案の量を増やしました。

    課題を分解し、体験を調整する

    テストプレイでは、感想をそのまま受け取るのではなく、何が体験を弱めているかに分解して改善します。違和感を言語化し、見た目・難易度・テンポなど、調整できる要素へ落とし込むことを重視しています。

    「ハムハム絵合わせ」では、達成感の薄さを「視覚的な迫力不足」と「後半の失速」に分解しました。ハムスターの密度・サイズ・速度変化を調整し、見た目の緊迫感を上げつつ、操作難度が急に跳ねないよう全体の速度も調整しました。
